Nodejs中使用路径别名

发表于2022-07-18|更新于2022-07-24|手记
|字数总计:620|阅读时长:2分钟|阅读量:

参考文章：React 之 config-overrides覆盖create-react-app配置、按需加载antd、Intro to the TSConfig Reference/#paths

问题描述

nodejs在导入模块的时候，使用相对路径的时候常常会出现模块嵌套过深，严重影响美观等一些其他的问题。

就比如import xxx from '../../../../xxx'

之前使用vue-cli脚手架的时候，脚手架是将相关的配置直接定义好的，比如导入src/component/xxx.vue时，可以简写成为@/component/xxx.vue，非常的美观，但是今天在使用create-react-app的时候，却发现脚手架并没有实现提供相关的配置。

本想着找到配置文件修改一下，但前端知识薄弱的我面对和vue-cli结构不太一样的项目，一下子就不会了。

不过还好，还有面向谷歌编程 😏

解决方案

首先，按照typescript文档，编辑tsconfig.json文件

添加以下这几行

"baseUrl": ".",
"paths": {
  "@/*": ["src/*"],
}

也可以不加baseUrl

"paths": {
  "@/*": ["./src/*"],
}

做到这一步，编辑器就可以识别了，但是很遗憾，webpack还是识别不了

然而，webpack配置在create-react-app脚手架中被隐藏了，想要修改配置必须进入node_modules目录，这显然不是一个优雅的解决方案。

想要优雅的话，我们需要导入以下两个包

yarn add react-app-rewired customize-cra -D

修改package.json里的启动配置（修改后）
