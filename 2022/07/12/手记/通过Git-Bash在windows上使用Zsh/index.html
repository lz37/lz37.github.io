通过Git Bash在windows上使用Zsh

发表于2022-07-12|更新于2022-07-24|手记
|字数总计:740|阅读时长:3分钟|阅读量:

参考: Install Terminal + git-bash + zsh + oh-my-zsh on Windows 10

猛然发现，Git Bash 某种程度上就是 bash，顿时喜上眉梢，这还用 powershell 干嘛呢，又慢又僵硬，果断换成使用 Git Bash。

安装 git bash for windows

https://git-scm.com/download/win

略

安装 zsh

下载链接↓\downarrow↓

https://packages.msys2.org/package/zsh?repo=msys&variant=x86_64

解压到 git 安装目录下，与之前的文件进行合并

安装 Oh My Zsh

执行(三选一)

如果显示连接问题的话，那就把https://raw.githubusercontent.com/ohmyzsh/ohmyzsh/master/tools/install.sh下载到本地执行

oh my zsh 相关的文件存放在 ~/.oh-my-zsh 目录下。

配置主题

我使用的主题名为powerlevel10k，使用方式如下

git clone --depth=1 https://gitee.com/romkatv/powerlevel10k.git ${ZSH_CUSTOM:-$HOME/.oh-my-zsh/custom}/themes/powerlevel10k

在 ~/.zshrc 中 设置 ZSH_THEME=“powerlevel10k/powerlevel10k”

要显示好看的主题别忘了安装必要的字体

配置插件

先贴一下我使用的插件

其中zsh-autosuggestions和zsh-syntax-highlighting是需要额外下载的插件，另外三个是自带的。

由于本人使用scoop包管理器，额外下了个windows的sudo命令(其实就是简单的让此行命令使用管理员权限运行而已)，所以添加了个sudo插件，双击esc即可在行前添加/减少一个sudo。

下面介绍需要下载的插件

zsh-autosuggestions 自动补全

https://github.com/zsh-users/zsh-autosuggestions

git clone https://github.com/zsh-users/zsh-autosuggestions ${ZSH_CUSTOM:-~/.oh-my-zsh/custom}/plugins/zsh-autosuggestions

之后编辑~/.zshrc文件即可

zsh-syntax-highlighting 语法高亮

git clone https://github.com/zsh-users/zsh-syntax-highlighting.git ${ZSH_CUSTOM:-~/.oh-my-zsh/custom}/plugins/zsh-syntax-highlighting

给history命令增加时间

编辑~/.zshrc文件，找到下面一行解除注释

HIST_STAMPS="mm/dd/yyyy"

更新 oh-my-zsh

默认的话是有更新的时候会提示，问你要不要更新。

如果需要它自动更新的话，把 ~/.zshrc 里下面这行注释取消就好了。

DISABLE_UPDATE_PROMPT=true

手动更新的话，执行omz update

解决中文无法显示的问题

编辑 C:/Program Files/Git/etc/bash.bashrc, 添加内容

export LC_ALL=en_US.UTF-8
