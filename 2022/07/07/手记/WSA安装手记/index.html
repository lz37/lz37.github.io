WSA安装手记

发表于2022-07-07|更新于2022-07-24|手记
|字数总计:755|阅读时长:2分钟|阅读量:

参考: Windows 11 安裝 Google Play 商店教學，WSA 一鍵腳本

WSA 简介

WSA 即 Windows Subsystem for Android™️, Windows 官网上是这么介绍的:

适用于 Android™️ 的 Windows 子系统使您的 Windows 11 设备能够运行 Amazon Appstore 中提供的 Android 应用程序。Android 是 Google LLC 的商标。如果您是一名有兴趣针对 Windows 桌面设备并针对 Windows 操作系统进行优化的开发人员，那么本指南适合您。

Windows 官方提供的的 WSA 只支持 Amazon Appstore, 但稍微捣鼓过安卓的应该有感觉, 没有谷歌框架的安卓是没有灵魂的, 更何况, Amazon Appstore 不支持国内用户 (虽然谷歌也不支持(>▽<)), 上面的应用数量也是少得可怜。

WSA 系统要求

  1. CPU：Intel Core i3 8th Gen (minimum) or above，AMD Ryzen 3000 (minimum) or above，Qualcomm Snapdragon 8c (minimum) or above
  2. 内存：8 GB 最低（不够用），建议 16 GB 及以上
  3. 磁盘：SSD，推荐 NVME SSD
  4. 可能需要为电脑的 BIOS/UEFI 启用虚拟化（现在一般默认启用）。

准备工作

点击开始菜单搜索并打开 “启用和关闭 Windows 功能”，勾选 “虚拟机平台” 点击 “确定” 即可，如图

专业版用户建议再勾选"Hyper-V"

大致操作

不过 github 上的大神LSPosed提供了一个免 wsl 安装谷歌框架，并且自带 Magisk 的 WSA 包，免去了手动更改官方包的痛苦。

而GDaily大佬基于此编写了一个一键完成的脚本，照着参考链接的操作一步步做就可以，十分简单，在此感谢两位大佬。

注意：如果先前没有安装过 WSA 的话，这一步有可能失败，原因很有可能是缺少相关依赖，但是一键脚本看不到报错，这里建议先安装一次普通的 WSA，看看报错。

普通的安装方式参照此链接：Windows 11 运行安卓子系统安装教程

安装好 WSA 之后，即可通过谷歌商店安装软件，如果要安装额外 APK 包的话，须先在 windows 上安装 ADB 并添加到 path 路径，WSA 上打开开发者模式然后经由 ADB 安装。

小白用户倘若不知如何操作的话，请打开命令行进行如下操作。

scoop会自动将 adb 添加到 path 中

后面就可以跳出命令行了，在微软商店中搜索WSA工具箱即可，WSA 工具箱会封闭命令行的运行情况，小白用户也能很快上手

最后，如果电脑需要长期跑 wsl 的（比如我），而且电脑只有区区 16G 内存的，还是打消这个念头吧，因为 WSA 打开自带 3 个 G，电脑风扇会狂转

还是装了之后才发现的问题，现在这 WSA 就像个鸡肋，食之无味，弃之可惜，哎。
