在Linux上优雅浏览Pixiv的姿势

发表于2022-05-05|更新于2022-07-24|工具
|字数总计:863|阅读时长:2分钟|阅读量:

简介：Pixiv是一个以插图、漫画和小说、艺术为中心的社交网络服务里的虚拟社区网站。于2007年9月10日推出第一个测试版。公司总部位于日本东京都涩谷区千驮谷。pixiv创办初衷是为全球艺术家提供一个能发表他们的作品，并透过评级系统反应其他用户意见的地方。网站以用户投稿的原创图画为中心，辅以标签、书签、作品回应、排行榜等功能形成具有其特色的社交网络。

省流：看图的网站，可以色色

需求的空缺

浏览Pixiv我喜欢用第三方的产品，毕竟Pixiv自身无论是APP还是网页端都实在是太拉垮了，就算使用一些扩展和脚本，其体验也是无法与第三方相比的。

Android上的Pixiv软件很多，譬如Pixiv-Shaft、Pix-EzViewer、PixivLite，各个用起来都很爽，尤其是前两者不分伯仲

Windows上的稍微少一些，最好用的个人感觉还是Pixeval，虽然不开代理的时候他的网络还是可能会有些许波动。

但在Linux上，这些工具都没有，那么该以什么样的姿势来访问P站呢？

（PixivBiu登录太过繁琐，不予考虑）

既然Linux上访问Pixiv如此之不便，那么，我不用Linux了，JOJO!

不过那是不可能的<(￣3￣)>

或许开个虚拟机，在虚拟机的平台里运行这些个软件，或许还可行？

但还是太过于繁琐，为了访问个Pixiv，至于么？还不至于为了大屏幕看图而牺牲这么多。

等下，安卓和Linux一脉相承，两者之间联系千丝万缕，不用虚拟机，直接在安卓容器里搞个呗！

好在还真可行，Anbox就是基于容器技术开发出来的一款安卓模拟器，可以比较大限度地模拟安卓的同时，资源开销方面也远远的低于虚拟机一大截。

具体的可以看我之前的一篇博文：Ubuntu20.04安装Anbox

具体操作

接下来将将怎么在Anbox里的具体操作。

首先安装好GooglePlay之后，使用GooglePlay安装Shaft。

登录Pixiv，如果没有出现文件名和密码的输入框的话，在手机上的Shaft的账号管理界面，将个人信息复制到剪切板，传到电脑上，然后点击登录界面“从剪切板导入”。

之后即可成功进入Shaft。

下载图片与下载管理

如何优雅地管理下载的插画，这是一个问题。

倘若想把插画批量地导入到本地，还得使用下面的命令

adb pull /storage/emulated/0/Pictures/ShaftImages/. /path/to/save/

十分不便

不过Anbox（snap商店版本）的储存目录在本地是有映射的（不如说就是储存在本地的），具体目录为/var/snap/anbox/common/data/media/0 ，对应的正好是/storage/emulated/0，之后的路径便是和安卓系统里相对应的了。

既然知道了具体的路径，那么正好，我TM直接一个ln…

ln -s /var/snap/anbox/common/data/media/0/Pictures/ShaftImages ~/Pictures/ShaftImages

用的是软连接，不占储存，还方便查看。

顺便再下一个

sudo apt install gnome-sushi

功能类似Windows上的QuickLook

再下一个

sudo apt install geeqie

好用的图片查看器
