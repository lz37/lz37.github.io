docker安装生产力工具记录

发表于2022-05-06|更新于2022-07-24|手记
|字数总计:766|阅读时长:3分钟|阅读量:

之前一段时间电脑经历了多次的重装，很多东西都丢失了，其中就包括了Docker的一些容器，于是各种容器全都得要重新部署。可是由于之前的部署过程并没有作详细的记录，导致一些东西都忘了，该踩的坑全都重新踩了一遍。

痛定思痛，是时候拿起键盘做一番记录了。

MySQL

拉取镜像

sudo docker pull mysql(:tag)

tag为指定的版本号，如果没有指定，则默认为latest，这里就直接用latest了。

布置volume

我的方法比较笨，先用以下命令运行一个容器

docker run --name mysql \
    -e MYSQL_ROOT_PASSWORD=password \
    -p 3306:3306 --restart=always -d \
    mysql

需要的数据卷cp出来

docker cp mysql:/etc/mysql /home/zerozawa/.config/mysql
docker cp mysql:/var/lib/mysql /home/zerozawa/database/mysql

将容器删除

docker rm -f mysql

重新运行容器

docker run --name mysql \
    -e MYSQL_ROOT_PASSWORD=password \
    -v /home/zerozawa/.config/mysql:/etc/mysql \
    -v /home/zerozawa/database/mysql:/var/lib/mysql \
    -p 3306:3306 --restart=always -d \
    mysql

这样，一个MySQL容器就创建好了，但是在容器外的连接还是会有问题

进入容器

docker exec -it mysql mysql -uroot -ppassword

依次运行

使用DBeaver测试连接

大功告成

Redis

在 https://github.com/redis/redis-hashes 上找到对应的源代码版本，下载

提取压缩包里的redis.conf文件，到个人指定的目录下（我自己指定的是/home/zerozawa/.config/redis/redis.conf）

选择一个容器里的路径，这个路径可以任选，比如我的是/etc/redis/redis.conf，但注意此路径要和待会填在redis-server后的要一致。

修改配置文件里的的一些内容：

bind 127.0.0.1 #注释掉这部分，使redis可以外部访问
daemonize no #用守护线程的方式启动
requirepass 你的密码 #给redis设置密码
appendonly yes #redis持久化　　默认是no
tcp-keepalive 300 #防止出现远程主机强迫关闭了一个现有的连接的错误 默认是300

千万注意，如果要写注释的话，注释请换行，别问我会有什么后果，出了事以后log一下你就懂了。

docker run \
    --name redis -d -p 6379:6379 --restart=always \
    -v /home/zerozawa/database/redis:/data \
    -v /home/zerozawa/.config/redis/redis.conf:/etc/redis/redis.conf \
    redis redis-server /etc/redis/redis.conf --appendonly yes

使用redis-desktop-manager连接一下做个验证

成功！

Nginx

docker run \
    --name nginx -p 80:80 -d \
    -v /home/zerozawa/.config/nginx/config:/etc/nginx \
    -v /home/zerozawa/.config/nginx/html:/usr/share/nginx/html \
    -v /home/zerozawa/.config/nginx/log:/var/log/nginx \
    nginx

修改挂载目录下的html文件，随便整点效果

OK，说明容器的运行和数据卷的挂载都没什么问题

Nacos

docker run --name nacos -e MODE=standalone -p 8848:8848 -d nacos/nacos-server

能访问IP:8848/nacos就说明成功（默认用户名和密码均是nacos）
